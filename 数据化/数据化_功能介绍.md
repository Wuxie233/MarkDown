### 断链法优化

> **设计思路**：传统数据化系统的一个普遍问题是"全部指令时刻运行"，即使某个模块当前不需要执行，相关的命令方块链也在每 tick 空跑，白白消耗服务器计算资源。断链法的核心理念是 **"满足条件再执行"**——将命令链在空闲时物理断开，仅在触发条件成立时才激活相关模块，从根本上消除无效计算。这样做的好处是：纯 RPG 服无需担忧系统造成卡顿问题。

### 玩家属性

大体可分为两类：**个人成长**和**战斗属性**。

#### 个人成长

| 属性 | 说明 |
|------|------|
| 玩家等级 | 线性天赋成长，每次升级 +10 下次升级所需；每次升级 +3 天赋点（175 级后不再给天赋点，425 级可点满全部天赋） |
| 怪物击杀数 | 记录击杀的怪物总数 |
| 玩家击杀数 | 记录击败的玩家总数 |
| 伤害统计 | 统计近战与射击造成的所有伤害 |
| 受伤统计 | 统计近战和射击所受的所有伤害 |

#### 战斗属性

> **设计思路**：战斗属性采用 **"基础 + 百分比"** 的双层叠算架构。每一项属性都由初始值、天赋加成、装备加成等基础数值先行累加，再乘以百分比增幅。装备的百分比加成为**独立乘区**，与天赋加成不会互相稀释。这种分离设计使得属性数值既有可预期的线性成长，又有装备带来的质变跃升。
>
> 直接修改攻击力、防御力等面板值会被刷新覆盖——需要修改的是对应的**初始属性**计分板（输入 `属_` 可以看到全部初始属性）。

- **生命值** = 初始生命值 + 天赋生命值 + 防具生命值，再乘以防具生命率
- **攻击力** = 初始攻击力 + 天赋攻击力 + 武器攻击力，再乘以武器攻击率
- **防御力** = 初始防御力 + 天赋防御力 + 防具防御力，再乘以防具防御率

$$防御率 = \frac{防御力}{防御力 + 300} \times 100\%$$

- **穿透力** = 武器穿透力

$$穿透率 = \frac{穿透力}{穿透力 + 300} \times 100\%$$

> **设计思路（防御率与穿透率公式）**：防御率和穿透率都使用了 **"数值 / (数值 + 常数)"** 的渐近收益公式。这意味着属性永远不会达到 100%，前期堆叠收益高、后期收益递减，从而避免了"堆满后无敌"的极端情况。常数 300 是经过反复调试选定的平衡点——让中等数值玩家的防御率落在 30%~50% 之间的体感区间。

- **闪避率** = 初始闪避率 + 天赋闪避率 + 护符闪避率
- **准确率** = 天赋准确率 + 护符准确率
- **暴击率** = 初始暴击率 + 武器暴击率 + 护符暴击率
- **爆伤率** = 初始爆伤率 + 武器爆伤率 + 护符爆伤率
- **回血率** = 基础回血率 + 等级回血率 + 护符回血率

$$等级回血率 = \frac{玩家等级}{10}（上限 10）$$

- **吸血率** = 武器吸血率
- **护盾值** = 当前护盾
- **经验** = 玩家等级 / 玩家经验 / 升级所需

#### 服务器货币

| 货币 | 说明 |
|------|------|
| Coin（硬币） | 服务器主要的交易货币，通过打怪等各种渠道获取 |
| Gold（金币） | 服务器最珍贵的交易货币，通过赞助支持或作为奖励少量获取 |

#### 对接接口

> **设计思路**：接口化设计是为了让系统使用者尽可能简单地与自己的其他系统对接。只需设定对应计分板值，系统会自动处理 UI 更新和倍率计算，省去手动调用多个命令的麻烦。

| 接口计分板 | 功能 |
|------------|------|
| `ad_coin` | 加硬币（享受倍率计算加成） |
| `ad_xp` | 加经验（享受倍率计算加成） |
| `ad_gold` | 加金币 |
| `ad_sw` | 加声望 |
| `ad_hp` | 加当前血量（通过该接口获得的血量恢复不会超出最大生命值） |

### 伤害计算

#### 闪避率与准确率

> **设计思路**：闪避和准确采用 **"对抗抵消"** 模型——实际闪避率 = 受击者闪避率 - 攻击者准确率。这使得闪避不是单方面的无解防御，而是一个可以被攻击者通过堆准确率来反制的机制。可以大于 100% 的设计则留出了"以高准确率碾压高闪避"的可能，避免数值天花板导致对抗博弈消失。

- 攻击时，实际闪避率 = 受击者闪避率 - 攻击者准确率
- 当准确率 > 闪避率时，受击者受到的任何伤害都不会触发闪避
- 示例：受击者闪避率 100%，攻击者准确率 80%，则受击者在受到该攻击者伤害时仅有 20% 的闪避率
- 准确率和闪避率的数值可以 > 100%

#### 回血相关

- 制作回血技能时可以调用内置的回血接口 `ad_hp`
- 直接设置值即可，通过回血接口回的血量不会超过玩家当前血量上限
- 当玩家满血时，调用该接口也不会使当前血量突破血量上限
- 玩家的等级回血率 = 玩家等级/10（最高 10%）
- 通过 [回血率] 属性恢复血量的频率直接改命令方块的延迟即可

#### 暴击相关

- 来自怪物的伤害也可以触发暴击（后台可设置怪物触发暴击的暴击率）
- 爆伤的乘区是在**全部削减伤害计算后最后乘的**

#### 伤害计算顺序

> **设计思路**：伤害计算采用**严格的流水线顺序**，每一步的输出作为下一步的输入。这种顺序设计保证了属性之间不会产生歧义——玩家和开发者都能准确预判一次攻击的最终伤害。特别是护盾和吸血放在最后，确保了"护盾完美格挡则不触发吸血"这一符合直觉的行为。

1. **UID 计算**：判断伤害来源。怪物受到的伤害能精确判定到攻击的玩家；玩家受到的伤害在有 UID 数据时精确判定，若无 UID 数据则来源为最近目标（一般是怪物攻击或玩家直接攻击）
2. **实际闪避率计算**：实际闪避率 = 受击者闪避率 - 攻击者准确率
3. **闪避判定**：根据实际闪避率随机判定是否闪避
4. **实际防御率计算**：实际防御率 = 受击者防御率 - 攻击者穿透率
5. **防御计算**：伤害 = 攻击力 × (100% - 实际防御率)
6. **免伤计算**：伤害 = 上一步伤害 × (100% - 受击者免伤率)
7. **暴击判定**：根据攻击者暴击率随机判定
8. **爆伤计算**：伤害 = 上一步伤害 × (100% + 攻击者爆伤率)
9. **护盾计算**：消耗受击者当前护盾值抵挡伤害
10. **吸血计算**：吸血量 = 穿过护盾的剩余伤害 × 攻击者吸血率（护盾完全格挡则无吸血；吸血不会超过血量上限）

### 等级机制

#### 初始等级

- 玩家初始等级为 1
- 1 级自带 3 天赋点
- 初始升级所需为 100
- 每次升级 +10 下次升级所需

#### 关于升级

> **设计思路**：升级检测采用**事件驱动**而非轮询——只有当全服范围内有任一玩家获得经验时才进行一次全服升级检测，而不是每 tick 检测。这大幅降低了无谓的计算消耗。一次检测至多处理 1 次升级，溢出的经验会保留用于下一次升级而不是清空，确保不浪费任何经验。

- 升级后玩家的经验条等级会同步为玩家当前等级，方便快速查看自身等级（使用**二分法** 在 1gt 内处理）
- 175 级及以前升级都能获得 3 天赋点（175 级时可将天赋全部点满）
- 溢出的经验会保留用于下一次升级

#### 调试工具

| 命令 | 说明 |
|------|------|
| `tag add 升级检测` | 进行一次全服范围的升级检测 |
| `tag add 玩家升级` | 进行全服升级检测时，拥有该标签的玩家可无条件升级 |

### 怪物等级

> **设计思路**：怪物等级的动态生成算法解决了一个经典问题——如何让不同进度的玩家在同一世界中都有合理的战斗体验。当怪物生成时：若周围有玩家，等级取周围全部玩家等级的平均值；若无玩家，在 1~100 范围内随机。这意味着新手区不会出现满级怪物碾压新手，而高等级玩家身边也不会全是低级白给怪，做到了**区域难度自适应**。

- 怪物等级无上限限制
- 怪物等级影响属性和奖励，但**不影响战利品的产出概率**（战利品产出概率默认 30%）

| 数值影响 | 公式 |
|----------|------|
| 属性提升 | 基础数值 × (1 + 怪物等级 × 1%) |
| 奖励提升 | 基础数值 × (1 + (怪物等级 ÷ 10 × 10)%) |

> 示例：怪物等级 = 37 → 全属性在基础上提升 37%，硬币与经验奖励在基础上提升 30%

### 战利品机制

> **设计思路**：传统打怪系统是怪死了直接给击杀者奖励，但这带来了两个问题：① 大佬带新手时秒杀怪物，新手什么都得不到；② 团队协作时无法分配奖励。战利品箱子机制通过一个中间步骤——**怪物死亡后生成掉落物宝箱，由玩家自行拾取获得奖励**——优雅地解决了这两个问题。

- 怪物死亡时不会直接给予奖励，而是生成一个**战利品宝箱**
- 玩家捡起宝箱时获得该怪物击杀应得的奖励
- 任何玩家拾取一个箱子后，周围的玩家都会获得该奖励 **10% 的分成**（人越多一起刷怪，击杀单个怪的总收益越多）
- 战利品箱子有存在时间限制，超时后自动清除
- 拾取箱子时有默认 30% 概率获得战利品道具（每种怪物可自定义独立掉落概率）
- 战利品道具存放在木桶里，使用**二分法免举**，无需添加额外 clone 等指令
- 怪物参数 `掉落方式`：为 0 时掉落木桶里全部道具，为 1 时掉落木桶里随机一个道具

### 战斗系统

#### 近战攻击

- 丢雪球触发挥砍攻击
- 雪球在装备武器后自动锁定在最后一个快捷栏

> **设计思路**：挥砍攻击设有随机攻击间隔（至多不超过 1s），在挥砍过快时会提示。这一设计有两个目的：① 降低服务器瞬时运算量，防止与其他系统在同一 gt 内大量计算引起延迟；② 增加战斗节奏感，避免无脑高速点击。

- 武器可以改变挥砍特效——理论上每款武器都能拥有独一无二的挥砍特效
- 攻击造成伤害或受到伤害时进入 **战斗状态**
- 近战伤害统计到 `伤害统计` 计分板
- 攻击内嵌 **UID 计算**，精确判定攻击者
- 攻击后，被攻击者的 `武器技能` 计分板值 = 攻击者的 `玩家武器` 值（可用于做武器技能触发和特定武器检测）
- 近战攻击**无法绕过护盾**——有护盾时优先对护盾造成伤害；护盾完全格挡则不触发吸血；伤害 > 护盾时护盾破损，仅造成溢出部分的伤害
- 近战攻击可以触发**吸血**（通过吸血获取的血量不会突破血量上限；满血时不触发吸血）
- 支持**多怪物/玩家在同一 gt 内同时进行受击计算**
- 武器**挥砍冷却**可自定义：冷却值为 0 时每次挥砍间隔 4~12 随机取值；小于 0 时无冷却

**对抗模式**：玩家间 PVP 需要双方都在战斗菜单中开启对抗模式，否则无法造成任何伤害。

#### 远程攻击

> **设计思路**：远程攻击在伤害模型上做了一个关键的差异化——弓箭的伤害是**穿透伤害**，能绕过实体的护盾直接造成伤害，但也因此失去了吸血的权利。（v260322 更新后弓箭伤害已统一走伤害引擎，护盾和吸血均生效）

- 射实体箭自动转换为数据化箭（可通过 summon arrow 做仿枪械）
- 装备的护符可以增添/改变弓箭的拖尾特效（提供接口参数）
- 弓箭穿墙是被允许的
- 弓箭受击检测内嵌 UID 计算，精确判定射击者
- 弓箭输出伤害与**射箭瞬间**玩家的属性面板有关
- 射箭伤害统计到 `伤害统计` 计分板

#### 战斗面板

> **设计思路**：战斗面板在战斗状态期间以 actionbar 形式持续显示关键信息，帮助玩家在战斗中实时掌握自身和目标的状态。

- 攻击/受击后进入**战斗状态**（持续 8 秒，期间提供抗性保护、不受原版伤害）
- 战斗状态下有三种面板模式：
  - **目标面板（怪物）**：附近 7 格有怪物时，显示怪物等级/名称/血条/属性 + 自身状态摘要
  - **目标面板（PvP）**：附近有其他战斗中玩家时，显示对方面板
  - **无目标面板**：附近无目标时，显示自身血条 + 武器/防具/护符技能冷却状态
- 血条使用 9 级阈值渐变显示，低血量（< 33%）红色，中等（34%~66%）橙色，高血量（67%~100%）绿色

### 伤害引擎接口

统一伤害结算引擎（3-5伤害引擎），处理闪避→防御→免伤→暴击→护盾→吸血→扣血→UI反馈完整流程。

#### 简化调用（推荐）

只需 3 步，引擎自动从攻击者读取准确/穿透/暴击/暴倍面板属性：

```
# 1. 设置伤害值到目标
scoreboard players set @e[目标选择器] 结算伤害 <伤害值>

# 2. 标记目标
tag @e[目标选择器] add 伤害结算

# 3. 标记攻击者
tag @s add _结算来源
```

引擎会在下一次处理时自动填充攻击者的面板属性，走完整流程。

#### 完整调用

手动指定全部参数，适合需要自定义属性值的场景：

```
# 在目标身上设置 5 个计分板
scoreboard players set @e[目标] 结算伤害 <伤害值>
scoreboard players set @e[目标] 结算_准确 <准确率>
scoreboard players set @e[目标] 结算_穿透 <穿透率>
scoreboard players set @e[目标] 结算_暴击 <暴击率>
scoreboard players set @e[目标] 结算_暴倍 <暴击倍率>  # 最低100(=×1)

# 标记接口
tag @e[目标] add 伤害结算
tag @s add _结算来源
```

> **判定规则**：引擎通过 `结算_暴倍` 是否为 0 判断是否需要自动填充。简化调用时不设置暴倍（默认 0），引擎自动读取；完整调用时暴倍 ≥ 100，引擎跳过自动填充。

#### 纯伤害（无攻击者）

如果不需要闪避/暴击等计算（如环境伤害、毒伤），可以不标记 `_结算来源`：

```
scoreboard players set @e[目标] 结算伤害 <伤害值>
scoreboard players set @e[目标] 结算_暴倍 1  # 非0即跳过自动填充
tag @e[目标] add 伤害结算
# 不添加 _结算来源 → 闪避/暴击/吸血全部跳过
```

### 战利品机制

> **设计思路**：传统打怪系统是怪死了直接给击杀者奖励，但这带来了两个问题：① 大佬带新手时秒杀怪物，新手什么都得不到；② 团队协作时无法分配奖励。战利品箱子机制通过一个中间步骤——**怪物死亡后生成掉落物宝箱，由玩家自行拾取获得奖励**——优雅地解决了这两个问题。老手可以选择不拾取让新手捡，团队也有了协商分配的余地。

- 怪物死亡时不会直接给予奖励，而是生成一个**战利品宝箱**
- 玩家捡起宝箱时获得该怪物击杀应得的奖励
- 任何玩家拾取一个箱子后，周围的玩家都会获得该奖励 **10% 的分成**（人越多一起刷怪，击杀单个怪的总收益越多）
- 战利品箱子有存在时间限制，超时后自动清除
- 拾取箱子时有默认 30% 概率获得战利品道具（2023.01.08 更新后每种怪物可自定义独立掉落概率）
- 战利品道具存放在木桶里，使用**二分法免举**，无需添加额外 clone 等指令

### 战斗系统

#### 近战攻击

- 丢雪球触发挥砍攻击
- 雪球在装备武器后自动锁定在最后一个快捷栏

> **设计思路**：挥砍攻击设有随机攻击间隔（至多不超过 1s），在挥砍过快时会提示。这一设计有两个目的：① 降低服务器瞬时运算量，防止与其他系统在同一 gt 内大量计算引起延迟；② 增加战斗节奏感，避免无脑高速点击。

- 武器可以改变挥砍特效——理论上每款武器都能拥有独一无二的挥砍特效（建议为高品质武器制作特殊特效；也可按武器类型分类，如剑类、镰刀类各一种）
- 攻击造成伤害或受到伤害时进入 **[战斗状态]**
- 近战伤害统计到 [伤害统计] 计分板
- 攻击内嵌 **UID 计算**，精确判定攻击怪物者
- 攻击后，被攻击者的 `武器技能` 计分板值 = 攻击者的 `玩家武器` 计分板值（可用于做武器技能触发和特定武器检测）

**对抗模式**：玩家间 PVP 需要双方都在战斗菜单中开启对抗模式，否则无法造成任何伤害。推荐使用雪球挥砍（附带攻击者 UID 数据，计算更精确）。

**近战的特殊机制**：
- 近战攻击**无法绕过护盾**——有护盾时优先对护盾造成伤害；护盾完全格挡则不触发吸血；伤害 > 护盾时护盾破损，仅造成溢出部分的伤害
- 近战攻击可以触发**吸血**（通过吸血获取的血量不会突破血量上限；满血时不触发吸血）
- 支持**多怪物/玩家在同一 gt 内同时进行受击计算**

#### 远程攻击

> **设计思路**：远程攻击在伤害模型上做了一个关键的差异化——弓箭的伤害是**穿透伤害**，能绕过实体的护盾直接造成伤害，但也因此失去了吸血的权利。这形成了有趣的战略取舍：近战有吸血续航但被护盾克制，远程无视护盾但没有吸血回复。

- 射实体箭自动转换为数据化箭（可通过 summon arrow 做仿枪械）
- 装备的护符可以增添/改变弓箭的拖尾特效（提供接口参数）
- 弓箭穿墙是被允许的（否则远程攻击使用率过低）
- 弓箭受击检测内嵌 UID 计算，精确判定射击者
- 弓箭输出伤害与**射箭瞬间**玩家的属性面板有关
- 射箭伤害统计到 [伤害统计] 计分板

#### 天赋菜单

> **设计思路**：天赋系统是数据化的核心长线玩法之一。天赋升级的属性提升不是固定值，而是在**指定范围内随机**，并引入了**幸运值**机制——提升越多，幸运值越高。这创造了一个"洗点博弈"的玩法循环：天赋升级提升的是基础数值，装备的百分比加成能吃到天赋带来的基础提升，因此玩家想要高面板就需要反复洗点去刷高的基础随机值。8 级攻击力天赋每次都 +6 的面板增益可能远超 15 级每次只 +3。

- 升级所需天赋点 = 所升属性当前的天赋等级
- **等级限制**：天赋等级最高 15 级（可自由关闭限制；tag=天赋突破 的玩家不受限制）
- **幸运值机制**：天赋升级时属性提升在指定范围内随机，幸运值根据实际提升量给予
- 提供天赋洗点接口：`tag add 天赋洗点`，执行后重置天赋属性并根据当前等级返还天赋点
